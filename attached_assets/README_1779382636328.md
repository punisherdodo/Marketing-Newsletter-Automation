# NovaMind AI-Powered Marketing Content Pipeline

## Overview

This project is a lightweight AI-powered marketing pipeline built for a take-home assignment for NovaMind, a fictional startup that helps small creative agencies automate daily workflows.

The app combines AI content generation, HubSpot CRM integration, persona-based segmentation, editable newsletter workflows, and performance-informed optimization in one Streamlit interface.

## What the system does

The pipeline supports the full campaign workflow:

- ideates campaign topics using AI
- generates a blog title, outline, draft, and persona-specific newsletters
- creates or updates contacts in HubSpot
- assigns contacts to personas
- maps the correct newsletter version to each persona segment
- logs campaign activity back to the CRM
- stores campaign content, revisions, and performance data locally
- generates recommendations for future campaigns

## Personas

The system segments contacts into five personas:

- Agency Founder
- Operations Manager
- Creative Lead
- Account / Client Services Lead
- Strategy / Marketing Lead

Persona assignment is rule-based and uses fields such as job title and related metadata.

## Architecture

The app is built as a local Streamlit interface with modular Python services.

Main components:
- `openai_service.py` for ideation, content generation, and optimization outputs
- `persona_service.py` for persona assignment
- `hubspot_service.py` for CRM sync, persona tagging, and campaign logging
- `storage_service.py` for SQLite storage and revision handling
- `analytics_service.py` for metric handling and weighted scoring
- `optimization_service.py` for persona ranking and recommendation formatting

Generated content is saved to JSON and SQLite. HubSpot is used as the CRM layer for contact storage, segmentation, and campaign logging.

## HubSpot integration

The system uses HubSpot to:
- create or update contacts
- store persona tags
- segment contacts by persona
- log campaign metadata at the contact level

The current prototype uses mock contacts, but those contacts live in HubSpot rather than only in local Python data.

### Example endpoint usage

The project uses realistic HubSpot CRM patterns, including:
- `POST /crm/v3/objects/contacts`
- `GET /crm/v3/objects/contacts/{email}?idProperty=email`
- `PATCH /crm/v3/objects/contacts/{email}?idProperty=email`
- `POST /crm/v3/properties/contacts`

### Example payload structure

Contact upsert payload:

    {
      "properties": {
        "email": "ops.demo@novamind-test.com",
        "firstname": "Liam",
        "lastname": "Cole",
        "company": "Northflow Creative",
        "jobtitle": "Operations Manager",
        "hs_persona": "Operations Manager"
      }
    }

Campaign log payload:

    {
      "properties": {
        "novamind_last_campaign_title": "AI in Creative Automation: Where Small Agencies Actually Save Time",
        "novamind_last_newsletter_variant": "Operations Manager",
        "novamind_last_send_date": "2026-04-14T16:02:04"

    {
      "to": "ops.demo@novamind-test.com",
      "persona": "Operations Manager",
      "newsletter_variant": "Operations Manager",
      "campaign_title": "AI in Creative Automation: Where Small Agencies Actually Save Time",
      "send_date": "2026-04-14T16:02:04"
    }

## Editing and optimization

The app allows users to edit the generated blog and newsletter drafts directly in the interface and save revisions.

After a campaign runs, the system generates:
- a performance summary
- persona-level recommendations
- subject line ideas
- next campaign options

The current weighted score formula is:

`0.3 * open_rate + 0.6 * click_rate - 0.1 * unsubscribe_rate`

The app also supports manual metric entry for updating campaign history.

## Real vs simulated

### Real
- AI topic ideation
- AI blog and newsletter generation
- HubSpot contact create or update
- HubSpot persona tagging
- HubSpot contact-level campaign logging
- local content editing
- revision saving
- local campaign and performance storage

### Simulated
- email delivery itself
- initial engagement metrics unless manually entered

This keeps the prototype runnable locally while still showing realistic CRM workflow design.

## Running the project locally

1. Install dependencies  
   `pip install -r requirements.txt`

2. Create a `.env` file in the project root with:
   - `OPENAI_API_KEY=your_openai_api_key`
   - `HUBSPOT_ACCESS_TOKEN=your_hubspot_private_app_token`
   - `OPENAI_MODEL=gpt-5.4`

3. Run the app  
   `python -m streamlit run app.py`

## Limitations

This is a working prototype rather than a production marketing platform.

Current limitations:
- the send layer is simulated rather than fully delivered through a live email platform
- performance analytics are only partially grounded in live outcomes unless manually entered
- optimization is primarily persona- and metadata-driven rather than full text-level content learning

## Final note

The main strength of this project is the integrated workflow: ideation, generation, segmentation, CRM sync, revision, distribution logic, and optimization in one interface.

A production extension would connect the same architecture to live email delivery and real engagement tracking.
